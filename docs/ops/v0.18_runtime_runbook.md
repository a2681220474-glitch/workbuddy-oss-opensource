# WorkBuddy OSS v0.18 Runtime Runbook

This runbook is for local and early private deployment operations. It focuses on
the `v0.18.x` runtime stack: API, Web, Feishu worker, WeCom callback, runtime
jobs, backups, and logs.

## Health Checks

- API health: `curl http://127.0.0.1:8000/health`
- Local services: `npm run services:status`
- Runtime stack: `npm run check:runtime-stack`
- Deployment readiness: `npm run check:deployment`
- Structured logs: `npm run check:logs`

## Backups

- Create backup: `npm run backup:create`
- Verify backup: `npm run backup:verify -- <backup-path>`
- Print restore plan: `npm run backup:restore-plan -- <backup-path>`
- Restore SQLite: `npm run backup:restore:sqlite -- <backup-path> --confirm`

Always create and verify a fresh backup before restore.

## Logs

- Log directory: `apps/api/data/logs`
- Supervised service logs: `apps/api/data/logs/services`
- Tail logs: `npm run logs:tail`
- Tail supervised output: `npm run services:logs`
- API log file: `api.jsonl`
- Feishu worker log file: `feishu-worker.jsonl`
- Runtime jobs log file: `runtime-jobs.jsonl`

Log fields are structured JSON Lines and redact keys that look like secrets,
tokens, passwords, or authorization headers.

Both structured logs and supervised process output rotate at 5 MiB by default
and retain three backups. Override with `WORKBUDDY_STRUCTURED_LOG_MAX_BYTES`,
`WORKBUDDY_STRUCTURED_LOG_BACKUPS`, `WORKBUDDY_SERVICE_LOG_MAX_BYTES`, and
`WORKBUDDY_SERVICE_LOG_BACKUPS`.

## Local Service Manager

- Start API, Web, and runtime jobs: `npm run services:start`
- Check health and restart counts: `npm run services:status`
- Restart managed services: `npm run services:restart`
- Stop managed services: `npm run services:stop`
- Start Feishu explicitly: `npm run services:start -- feishu-worker`

The default profile deliberately excludes the real Feishu worker.

## Runtime Jobs

- Check config: `npm run check:background-jobs`
- Run once: `ENABLE_BACKGROUND_JOBS=true .venv/bin/python -m apps.api.workers.runtime_jobs --once`
- Run continuously: `npm run dev:runtime-jobs`

The worker currently performs:

- approval delivery retry scan
- overdue task scan
- stale ticket scan
- scheduled report generation

## IM Acceptance

- Feishu stream config: `npm run check:feishu-stream`
- Feishu acceptance: `npm run check:feishu-acceptance`
- WeCom config: `npm run check:wecom-runtime`
- WeCom acceptance: `npm run check:wecom-acceptance`

## Docker Boundary

The target Compose stack is documented in `docker-compose.yml`, but this local
machine currently has no Docker command available. Full Compose boot must be
verified on a Docker-capable host with:

- `docker compose up --build`
- `npm run db:migrate`
- `npm run check:deployment`
