# Approval And Audit

WorkBuddy OSS treats human review and audit logs as product foundations, not enterprise extras.

## Approval Principle

Any customer-facing, candidate-facing or external group reply generated by an Agent must enter the approval queue first.

Phase 0 approval states:

- `pending`: waiting for human review
- `approved`: accepted by a human
- `edited`: modified before approval
- `rejected`: not allowed to send

Phase 0 does not actually send external messages. Approval only proves the control loop.

## Required Approval Context

An approval item should show:

- original message
- sender and conversation
- detected scenario and intent
- related Ticket or Lead
- Agent confidence
- draft reply
- structured Action JSON
- AgentRun link

## AgentRun Audit

Every Agent behavior should produce an AgentRun record, including mock/demo decisions.

Required audit goals:

- know which message triggered the Agent
- know which Agent handled it
- know which prompt/rule/model path was used
- know the structured Action that was proposed
- know whether an approval was created
- know whether the run failed

This design makes Phase 0 useful even before real IM integrations exist: every demo action can be inspected and replayed.
