# WorkBuddy OSS v1.0.1 ECS First Deploy Fix

`v1.0.1` fixes the first private deployment issue found during the initial
Alibaba Cloud ECS rollout.

## What changed

- The baseline Alembic migration no longer creates tables that belong to later
  knowledge-base revisions.
- The API image no longer runs migrations on every container boot.
- `deploy/oci-free/deploy.sh` now starts Postgres and Redis first, runs
  migrations once, and then starts API, web, runtime jobs, and Caddy.
- `deploy/oci-free/bootstrap_ubuntu.sh` now installs Docker Compose v2
  explicitly when Ubuntu does not provide the plugin package.
- Deployment docs and health expectations are updated to `1.0.1`.

## Why this was needed

The first fresh Postgres deployment hit a migration race and schema conflict:
the baseline migration created knowledge tables too early, then later Alembic
revisions tried to create them again. That left the API and runtime worker
restarting and the public endpoint returning `502 Bad Gateway`.

## Result

Fresh ECS deployments now use a single serialized migration path, and the ECS
host is aligned to Docker Compose v2 so later redeploys can recreate services
cleanly on the same server.
