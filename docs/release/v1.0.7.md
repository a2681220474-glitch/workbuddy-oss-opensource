# WorkBuddy OSS v1.0.7 Runtime Secret Safety Cues

`v1.0.7` clarifies how runtime secrets are handled in the configuration center.

## What Changed

- Updated configuration-center copy to explain that secrets are masked and not
  echoed back.
- Clarified that empty secret fields keep existing values unchanged.
- Preserved audit behavior that records key names and configuration actions,
  not secret values.

## Optimized

- Admins get clearer feedback before editing DeepSeek, Feishu, WeCom, DingTalk,
  and other runtime credentials.

## Fixed

- Fixed ambiguity around whether secret fields would reveal or overwrite stored
  values.

## Final Result

The configuration center is safer and easier to understand before the later
encrypted-secret storage milestone.

## Manual Acceptance

1. Open `配置中心` as an admin.
2. Confirm the notice says secrets are stored locally, masked, and not echoed.
3. Confirm configured secret fields say leaving them blank keeps existing
   values.
4. Save a non-secret value and confirm secrets are not displayed.
