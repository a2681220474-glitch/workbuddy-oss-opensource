# WorkBuddy OSS v1.0.5 Session Hardening and Favicon

`v1.0.5` improves the local login/session experience and adds a recognizable
browser tab icon.

## What Changed

- Added a `WB` SVG favicon that matches the sidebar brand mark.
- Added the favicon link to the web HTML entry.
- Added a frontend session-expired event when business API calls return `401`.
- Updated the app shell to clear local user state and return to login when a
  session expires.

## Optimized

- Browser tabs now show the WorkBuddy identity instead of the default globe.
- Expired sessions now produce a visible login-state recovery path.

## Fixed

- Fixed the app staying visually authenticated after a protected API call
  reports that the session is gone.

## Final Result

The local web app has a clearer product identity in the browser tab and a more
predictable session-expiration path.

## Manual Acceptance

1. Open `http://localhost:5173/`.
2. Confirm the browser tab shows the `WB` icon.
3. Log in, then clear the `workbuddy_session` cookie or let it expire.
4. Trigger a business API action and confirm the app returns to login.

## Verification

- `npm run build:web`
