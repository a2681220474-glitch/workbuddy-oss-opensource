# WorkBuddy OSS v0.20.2 Deployment Acceptance Prep

`v0.20.2` adds deployment documentation and Docker Compose acceptance guidance.

## What Changed

### Added

- Added `docs/DEPLOYMENT.md`.
- Added `docs/ops/docker_compose_acceptance.md`.
- Extended `npm run check:release` to require deployment acceptance docs.

### Optimized

- Deployment docs now separate local SQLite development from the
  Docker/PostgreSQL/Redis release-candidate target.
- Docker Compose acceptance has a concrete static check, boot check, runtime
  check, demo workflow check, backup check, and shutdown step.
- Backup and restore commands are linked from the deployment guide.

### Fixed

- Compose readiness is no longer only implied by `docker-compose.yml` and older
  runtime notes.
- The docs explicitly state that static checks do not prove a real Docker boot
  when Docker is unavailable on the current machine.

## Final Result

The release candidate now has an explicit deployment path:

```text
DEPLOYMENT -> docker_compose_acceptance -> check:deployment -> runtime checks
```

## Manual Acceptance

Run:

```bash
npm run build:web
.venv/bin/python -m compileall apps/api scripts/check_release_readiness.py
npm run db:migrate
npm run check:deployment
npm run check:release
```

On a Docker-capable machine, additionally run:

```bash
docker compose config
docker compose up --build
```
