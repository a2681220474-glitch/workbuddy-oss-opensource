# WorkBuddy OSS v1.1.4 正式 RAG 基础能力

`v1.1.4` 完成九项后续基线中的第 7 项：知识检索从关键词匹配升级为可私有化运行的混合 RAG，并补齐引用、反馈、回滚和审计闭环。

## 本版做了什么

- 新增本地确定性向量索引，不依赖外部 embedding 服务、API Key 或调用费用。
- 将关键词得分、向量相似度和知识质量分组合为统一检索排序。
- 搜索页和 Agent 消息路由改为复用同一套检索服务。
- 新增知识索引迁移、重建接口和内容哈希失效机制。
- 新增知识命中反馈和历史版本回滚接口。

## 新增能力

- `KnowledgeEmbedding` 保存知识条目的本地向量、模型版本和内容哈希。
- `POST /api/knowledge/index/rebuild` 可重建当前租户索引。
- 检索结果返回综合分、关键词分、语义分、质量分、引用编号和证据片段。
- 引用统一使用 `[KB-{id}]`，可继续追溯到 KnowledgeItem 和 KnowledgeHit。
- 命中结果支持“有帮助/无帮助”反馈。
- 版本历史支持一键回滚，回滚后生成新的正式版本快照。
- 新增 `npm run check:rag-workflow`，在临时数据库完成 10 项隔离验收。

## 优化

- 中文检索加入字符片段与常见业务概念映射，可识别“登录失败”和“账户登不上”等近义表达。
- 证据片段优先选择与问题最相关的答案句段，而不是固定截取答案开头。
- 知识内容变化后会根据内容哈希自动刷新索引，避免旧向量继续参与排序。
- AgentRun 和审批上下文中的知识引用现在携带统一 citation 和语义分。
- 知识页“轻量检索”升级为“RAG 检索”，可直接查看排序构成。

## 修复

- 修复仅靠整词关键词时，中文近义问法容易完全漏检的问题。
- 修复搜索页与消息路由各自维护检索逻辑、结果可能不一致的问题。
- 修复检索命中没有人工有效性反馈、质量治理无法形成闭环的问题。
- 修复知识版本只能查看、不能恢复历史内容的问题。

## 最终结果

- 本地 RAG 不读取或上传 DeepSeek、飞书、企微等敏感配置。
- 现有知识、历史版本和命中记录均保留，新增索引为独立表。
- 负反馈会降低知识质量分并标记待复审；正反馈可逐步恢复质量分。
- 回滚、索引重建和命中反馈全部进入处理记录或审计总账。
- 10 项 RAG 隔离验收全部通过，前端生产构建通过。
- 远程 ECS 继续冻结在 `v1.0.1`，本版未部署远程。

## 人工验收

1. 打开 `http://localhost:5173/#knowledge`，进入“RAG 检索”。
2. 点击“重建索引”，确认提示已重建知识条目。
3. 输入与现有知识意思相近但字面不同的问题，确认结果显示 `[KB-id]`、综合分、关键词分、语义分和证据片段。
4. 点击“无帮助”，进入该知识详情，确认命中状态为 `not_useful`、质量状态变为待复审且质量分下降。
5. 编辑一条知识并保存，再打开详情的“版本历史”，点击旧版本“回滚”。
6. 确认答案恢复，版本历史顶部新增 `rollback` 版本。
7. 运行 `npm run check:rag-workflow`，确认 10 项全部通过。
