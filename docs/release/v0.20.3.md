# WorkBuddy OSS v0.20.3 Security And Open-Source Hygiene

`v0.20.3` adds the safety and repository hygiene layer needed before release
candidate finalization.

## What Changed

### Added

- Added `SECURITY.md`.
- Added `docs/PRIVACY_SECURITY.md`.
- Added `docs/RELEASE_HYGIENE.md`.
- Added ignore rules for local generated documents, demo video output, and the
  one-off video generation script.
- Extended `npm run check:release` to require security and hygiene docs.

### Optimized

- Security guidance now names secret paths, connector credentials, model
  provider risk, external send defaults, logs, backups, and redaction rules.
- Release hygiene now documents what must not be committed and what claims must
  not be made without matching verification.

### Fixed

- Local generated artifacts are less likely to be accidentally staged during
  later release work.
- Privacy and security notes are no longer scattered across connector docs and
  handoff notes.

## Final Result

The release candidate now has a safety layer:

```text
SECURITY -> PRIVACY_SECURITY -> RELEASE_HYGIENE -> check:release
```

## Manual Acceptance

Run:

```bash
npm run build:web
.venv/bin/python -m compileall apps/api scripts/check_release_readiness.py
npm run db:migrate
npm run check:release
git status --short
```

Then verify:

1. Security and privacy docs exist.
2. Local generated artifacts remain untracked and ignored.
3. `npm run check:release` passes.
4. Release docs do not claim unsupported production capabilities.
