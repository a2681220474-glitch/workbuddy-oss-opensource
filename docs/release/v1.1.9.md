# WorkBuddy OSS v1.1.9 深层反馈收口

`v1.1.9` 在 `v1.1.8` 主验收链路清理基础上，把深层业务页面和组件里的 Ant Design 静态反馈调用统一收口，避免后续页面交互再次出现上下文警告。

本版仍属于 `v1.1.x` 本地正式收口后的维护版本：只做稳定性、文档和验收一致性维护，不扩展产品面。

## 本版做了什么

- 将深层页面的静态 `message` 导入改为 `AntdApp.useApp()`。
- 将审批页、知识详情里的静态 `Modal.confirm/warning` 改为上下文 `modal`。
- 覆盖工作台、消息、审批、工单、线索、社群、任务、候选人、知识、报告、团队、诊断和业务对象详情抽屉。
- 升级 `npm run check:frontend-maintenance`，从主链路检查扩展为全量 `apps/web/src/**/*.tsx` 扫描。
- 更新 release、handoff、Known Issues、Obsidian 项目记忆和版本号。

## 新增能力

- 自动阻止新的静态 `message` 导入回流。
- 自动阻止新的静态 `Modal.confirm/warning/error/info/success` 调用回流。
- 自动检查所有反馈调用都处在 Ant Design `App` 上下文内。
- 发布前聚合命令 `npm run check:formal-release` 会继续执行该检查。

## 优化

- 深层页面交互的控制台体验更干净。
- 减少后续人工验收中因 Ant Design 上下文提示造成的噪音。
- 入口 bundle 继续低于 500 kB 预算。
- 继续保持本地优先，不触碰远程 ECS。

## 修复

- 修复深层页面仍可能触发 `[antd: message] Static function can not consume context` 的风险。
- 修复审批页和知识详情回滚确认仍使用静态 Modal helper 的风险。
- 修复前端维护检查只覆盖主链路、不覆盖深层页面的问题。

## 最终结果

- 本地版本推进到 `v1.1.9`。
- 全量前端反馈检查入口：`npm run check:frontend-maintenance`。
- 聚合发布检查：`npm run check:formal-release`。
- 远程 ECS 继续冻结在 `v1.0.1`，本版未部署远程。
- 本版未触发真实飞书、企微、钉钉外发。

## 人工验收

1. 打开 `http://localhost:5173/`。
2. 登录后确认顶部显示 `v1.1.9 深层反馈收口`。
3. 打开浏览器控制台。
4. 进入 `审批队列`、`知识库`、`消息事件`、`客服工单`、`配置中心` 等页面。
5. 触发一个无外发副作用的操作，例如筛选、查看详情、打开确认弹窗或 mock 测试。
6. 确认没有 `[antd: message] Static function can not consume context`。
7. 确认没有静态 `Modal` 上下文警告和 `destroyOnClose` 弃用提示。
8. 运行 `npm run check:frontend-maintenance`，确认全量检查通过。
9. 运行 `npm run check:formal-release`，确认聚合发布检查通过。

## 自动验证

```bash
npm run check:frontend-maintenance
npm run check:formal-release
npm run check:rc
npm run build:web
npm run check:web-bundle
.venv/bin/python -m compileall apps/api scripts
```

浏览器验收继续使用临时隔离数据库、临时认证密钥和禁止外发策略，不修改正式业务数据。
