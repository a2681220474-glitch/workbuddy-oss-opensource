# WorkBuddy OSS v0.13.0 Configurable Runtime

v0.13.0 shifts the project away from release packaging and back toward becoming a complete, locally usable product.

## What Changed

- Unified runtime configuration around `.env` and process environment variables.
- Added first-class LLM settings to backend `Settings`.
- Implemented real OpenAI-compatible chat completion calls for Router fallback classification.
- Kept deterministic rules-first scenario Agents so local workflows remain stable.
- Added Config Center visibility for model provider, model name, key readiness and channel setup state.
- Expanded Feishu status handling without forcing every test into mock-only mode.
- Added WeCom and DingTalk configuration status, webhook paths and local webhook import endpoints.
- Updated `.env.example` with real model/channel setup keys.

## Current Product Truth

This is not a finished production SaaS yet.

It is now a configurable local runtime with:

- complete local business-object flow
- four deterministic scenario Agents
- approval and audit flow
- real Feishu test capability when credentials are present
- OpenAI-compatible LLM provider hook
- WeCom/DingTalk setup and webhook entry points ready for test-account integration

Still pending:

- full model-enhanced scenario Agent reasoning
- WeCom/DingTalk platform signature verification and encrypted callback support
- WeCom/DingTalk real outbound send
- production permissions, tenants and deployment hardening

## Acceptance Checks

1. `cp .env.example .env`
2. Fill local Feishu or LLM credentials as needed.
3. Start API and Web.
4. Open Config Center and confirm model/channel status.
5. Open Demo Mode and run one-click Beta validation.
6. Optional: send test Feishu messages through the test app.
7. Optional: POST sample WeCom/DingTalk payloads to the new webhook endpoints.

## Validation Commands

```bash
.venv/bin/python -m compileall apps/api
npm run build:web
curl http://localhost:8000/api/config/status
```
