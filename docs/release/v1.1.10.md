# WorkBuddy OSS v1.1.10 轻量 UI 验收

`v1.1.10` 在本地正式收口和深层反馈收口之后，补上轻量 UI smoke 检查，把核心页面路由、懒加载页面文件、顶部版本、favicon 和发布文档一致性纳入自动验收。

## 本版做了什么

- 新增 `npm run check:ui-smoke`。
- 检查 23 个核心 Hash 路由是否都能映射到对应懒加载页面组件。
- 检查所有 `App.tsx` 懒加载页面导入是否有真实页面文件。
- 检查高频导航入口文案：工作台、消息事件、审批队列、知识库、配置中心、飞书诊断、企微诊断。
- 检查浏览器标签 favicon 是否继续使用 `WB` 品牌图标。
- 将 `check:ui-smoke` 并入 `npm run check:formal-release`。
- 更新 release、handoff、Known Issues、Obsidian 项目记忆和版本号。

## 新增能力

- UI 路由清单从人工记忆升级为脚本化验收。
- 发布前可以自动发现“菜单还在、页面文件丢了”或“版本显示与文档不同步”的问题。
- 聚合发布检查现在覆盖后端收口、前端反馈上下文、轻量 UI 路由、产品闭环、RAG、连接器安全、RC、构建和 bundle 预算。

## 优化项

- 减少后续小版本改页面时遗漏路由或懒加载文件的风险。
- 让 favicon、顶部版本号、release 文档和交接文档保持同一套验收口径。
- 保持远程 ECS 冻结，继续先本地跑通再决定是否部署。

## 修复项

- 修复发布验收中缺少轻量 UI 路由一致性检查的问题。
- 修复小版本文档和浏览器可见版本容易漂移的问题。

## 最终结果

- 本地版本推进到 `v1.1.10`。
- 新增检查入口：`npm run check:ui-smoke`。
- 聚合发布检查：`npm run check:formal-release` 已包含 UI smoke。
- 远程 ECS 继续冻结在 `v1.0.1`，本版未部署远程，也未触发真实外发。

## 人工验收

1. 启动本地服务：`npm run services:restart`。
2. 打开 `http://localhost:5173/`。
3. 登录后确认顶部显示 `v1.1.10 轻量 UI 验收`。
4. 确认浏览器标签显示 `WB` 图标，而不是默认小地球。
5. 依次访问工作台、消息事件、审批队列、知识库、配置中心、飞书诊断、企微诊断。
6. 页面切换过程中确认没有空白页、没有明显报错 toast、控制台没有新增阻塞错误。
7. 运行 `npm run check:ui-smoke`，确认全部通过。
8. 运行 `npm run check:formal-release`，确认聚合发布检查通过。
