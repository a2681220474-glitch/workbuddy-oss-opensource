# WorkBuddy OSS v0.18.0 Deployment Track Start

`v0.18.0` marks the end of the `v0.17.x` major line and the start of the
`v0.18.x` deployment, data, and background-task track.

## What Landed In v0.17.x

- Added a real WeCom channel router:
  - `GET /api/channels/wecom/status`
  - `GET /api/channels/wecom/diagnostics/full`
  - `GET /api/channels/wecom/webhook`
  - `POST /api/channels/wecom/webhook`
- Added WeCom callback URL verification, signature checking, AES decrypt, XML
  parsing, and JSON debug payload ingestion.
- Added WeCom `access_token` fetching and in-process caching.
- Added WeCom message normalization into `MessageEvent`.
- Added WeCom business trace, acceptance trace, and recent event/run diagnostics.
- Added WeCom delivery support inside approval send preview and approval send flow:
  - application user send
  - ChatId group send
- Added WeCom diagnostics UI page:
  - `http://127.0.0.1:5173/#wecom`
- Added CLI scripts:
  - `npm run check:wecom-runtime`
  - `npm run check:wecom-acceptance`

## Manual Acceptance

1. Open `http://127.0.0.1:5173/#wecom`.
2. Confirm the diagnostics page shows:
   - callback requirements
   - send requirements
   - production readiness
   - recent WeCom events
3. Open `http://127.0.0.1:5173/#channel-events` and confirm the imported WeCom
   event is visible.
4. Open `http://127.0.0.1:5173/#messages?id=57` and confirm the WeCom sample
   message exists.
5. Open `http://127.0.0.1:5173/#approvals?id=21` and confirm:
   - original message is WeCom
   - approval context shows the related lead
   - send preview resolves `channel=wecom`
   - target policy shows `target_type=chat` and the target id
6. When real WeCom credentials are available, save them in `#config`, run
   `npm run check:wecom-runtime`, then complete callback URL verification in the
   WeCom admin console.

## Notes

- `check:wecom-runtime` will fail until real `WECOM_*` credentials exist in the
  current machine's runtime config.
- `check:wecom-acceptance` now passes against the local WeCom debug sample and
  confirms a `ready` chain.
