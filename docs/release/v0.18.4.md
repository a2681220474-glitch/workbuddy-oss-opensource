# WorkBuddy OSS v0.18.4 Structured Logs And Runtime Runbook

`v0.18.4` continues the deployment and operations major line by adding
structured runtime logs and a practical runbook.

## What Landed

- Added structured JSON Lines logging:
  - API request lifecycle
  - API startup
  - Feishu worker check/start
  - runtime-jobs check/start/cycle/error
- Added log inspection script:
  - `scripts/check_logs.py`
  - `npm run check:logs`
  - `npm run logs:tail`
- Added log runtime status in `/health` and config status.
- Config Center now shows log readiness, log directory, recent log files, and
  the tail command.
- Added v0.18 operations runbook:
  - `docs/ops/v0.18_runtime_runbook.md`
- Deployment checks now include log readiness and compose service declarations.

## Verification

- `.venv/bin/python -m compileall apps/api scripts/check_logs.py scripts/check_deployment.py`
- `npm run check:logs`
- `npm run check:deployment`
- `curl http://127.0.0.1:8000/health`

## Manual Acceptance

1. Open `http://127.0.0.1:5173/#config`.
2. Confirm the deployment card shows log status and log commands.
3. Run `npm run check:logs`.
4. Run `npm run logs:tail`.
5. Confirm logs are written under `apps/api/data/logs`.
