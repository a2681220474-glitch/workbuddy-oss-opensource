# WorkBuddy OSS v1.0.2 Formal Local Login Start

`v1.0.2` moves the local product line from header-based identity switching to a
real login foundation.

## What changed

- Added `password_hash` storage for local users with Alembic migration
  `20260606_0005_local_auth_passwords.py`.
- Added one-time local auth bootstrap for the first admin password.
- Added login, logout, and change-password APIs with signed HttpOnly session
  cookies.
- Added an API auth gate so unauthenticated business requests return `401`.
- Added a dedicated login/bootstrap page in the web app.
- Updated the team page to set initial passwords for new users and reset
  passwords for existing users.

## Why this was needed

Before this version, WorkBuddy relied on browser-side local identity switching
and the `X-WorkBuddy-User` header. That was useful for early workflow
development, but it was not a real authentication layer and could not be
considered private-deployment ready.

## Result

The local product path now supports:

- first-time admin password initialization
- formal login and logout
- self-service password change
- authenticated access to the main business APIs

The remote ECS server remains frozen at `v1.0.1` by user decision; `v1.0.2`
currently advances the local-first mainline only.

## Manual acceptance

1. Open the local web app.
2. Confirm you land on the login/bootstrap screen instead of the old local user
   switch flow.
3. If this is the first run, initialize the local admin password.
4. Confirm `/api/users` returns `401` before login.
5. Login and verify the main workspace loads normally.
6. Open `团队成员` and create a new local user with an initial password.
7. Edit that user and confirm the password can be reset.
8. Use the top-right password action to change the current user's password.
9. Logout and login again with the new password.
