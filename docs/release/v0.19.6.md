# WorkBuddy OSS v0.19.6 Lightweight Knowledge Retrieval

`v0.19.6` continues the `v0.19.x` knowledge and RAG formalization line. This
release adds lightweight keyword retrieval as the bridge between the knowledge
repository and later Agent citations.

## What Changed

### Added

- Added `POST /api/knowledge/search`.
- Added keyword token scoring across title, answer, and category.
- Added quality-aware scoring penalties for expired, optimization-needed, and
  archive-suggested knowledge.
- Added optional retrieval hit recording through the existing `KnowledgeHit`
  table.
- Added the knowledge page `иЅ»й‡ЏжЈЂзґў` tab.
- Search results now include score, reasons, snippet, source reference, and
  recorded hit id.

### Optimized

- Knowledge search can filter by the currently selected category.
- Search records are deduplicated by query hash and source object metadata.
- The quality panel refreshes after recorded retrieval hits so governance can
  see retrieval activity.

### Fixed

- KnowledgeItem records are no longer only discoverable through table scanning;
  users can now test retrieval behavior before full RAG is introduced.

## Final Result

WorkBuddy now has a basic retrieval loop:

```text
query
  -> keyword scoring
  -> ranked KnowledgeItem matches
  -> source reference + snippet
  -> optional KnowledgeHit record
```

This is not vector RAG yet. It is the lightweight retrieval layer needed before
embedding indexes and Agent citation workflows.
