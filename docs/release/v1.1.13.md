# WorkBuddy OSS v1.1.13 飞书卡片回调兼容

`v1.1.13` 修复飞书审批卡片点击“通过”或“拒绝”后，审批状态已成功更新但飞书客户端仍提示 `code: 200672` 的问题。

## 本版做了什么

- 对照飞书官方 Python SDK 的卡片回调示例校正 HTTP 响应格式。
- 审批回调成功后直接返回更新后的完整卡片 JSON。
- 内部仍保留审批结果、提示文案和卡片回调运行记录。
- 连接器验收同时覆盖“通过”和“拒绝”两种完成态。

## 新增

- 新增飞书拒绝回调自动验收。
- 新增原始替换卡片协议检查，阻止 `toast + card` 包装格式回流。

## 优化

- 飞书客户端收到符合卡片回调协议的响应后，可直接刷新原卡片。
- 通过和拒绝使用同一套响应契约，避免两条分支表现不一致。

## 修复

- 修复审批已经写入成功但飞书仍显示 `200672`。
- 修复原卡片无法在点击后更新为“已通过/已拒绝 + 查看详情”。
- 修复自动验收只检查内部 JSON 内容、未检查真实飞书响应外形的问题。

## 最终结果

- 当前本地版本：`v1.1.13`。
- “查看详情”继续使用公网 URL，不经过卡片动作回调。
- 消息 ID 幂等、北京时间序列化和审批状态写入逻辑保持不变。
- 本地正式收口状态保持不变，真实外发仍需用户明确授权。
- 远程 ECS 后续不随每个小版本自动升级。
- 远程 ECS 已受控升级到 `v1.1.13`，公网健康检查和飞书 worker 长连接正常。
- 部署前 PostgreSQL 备份：`/path/to/backups/workbuddy-before-upgrade-YYYYMMDD-HHMMSS.sql`。
- 真实飞书通过/拒绝仍需用户用全新卡片完成最终复验。

## 人工验收

1. 在飞书发送一条全新测试消息，确认 WorkBuddy 只新增一条记录且时间为北京时间。
2. 对新审批卡片点击“通过”，确认不再提示 `200672`，原卡片变为“已通过 + 查看详情”。
3. 再生成一张新卡片并点击“拒绝”，确认不再提示 `200672`，原卡片变为“已拒绝 + 查看详情”。
4. 点击“查看详情”，确认正常打开线上对应审批。
