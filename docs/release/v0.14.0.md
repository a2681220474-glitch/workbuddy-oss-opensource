# WorkBuddy OSS v0.14.0 Agent Runtime

v0.14.0 turns configuration and routing into a usable local runtime instead of a read-only status screen.

## What Changed

- Config Center now has forms for LLM, Feishu, WeCom, DingTalk and runtime policy.
- Saving from the page writes to local `.env.local`; secrets are not returned to the browser.
- OpenAI-compatible LLM routing now records request metadata, usage, provider raw status and error reason.
- Low-confidence LLM classifications go to `manual_inbox_agent` instead of silently looking like normal chat.
- Rule-first routing still protects the four business agents: support, sales, community and recruiting.

## Manual Acceptance

1. Open `http://localhost:5173/#config`.
2. Save a model provider, base URL, API key and model from the "大模型配置" form.
3. Refresh Config Center and confirm "真实 LLM" changes to configured when base URL, API key and model are present.
4. Save Feishu App ID / Secret from the "飞书配置" form, then open Feishu diagnostics and run token check.
5. Import or send a message that matches no strong keyword rule and confirm the resulting AgentRun uses `manual_inbox_agent` when model confidence is low or the model call fails.
6. Send a clear sales/support/community/recruiting message and confirm rule-first routing still creates the expected business object.

## Notes

- The API process reloads config immediately after saving from Config Center.
- The Feishu stream worker is a separate process and must be restarted after Feishu credential changes.
- WeCom and DingTalk still need platform signature/decryption and real send integration with test accounts.
