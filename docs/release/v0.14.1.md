# WorkBuddy OSS v0.14.1 Runtime Repair Console

v0.14.1 focuses on making real local testing less brittle. When a historical
message was classified by older rules, operators can now rerun that message from
the message event table and regenerate the current business objects.

## Changes

- Added `POST /api/messages/{message_id}/rerun`.
- Rerun replaces the message's auto-generated tickets, leads, tasks, candidates,
  knowledge gaps, and pending approvals before creating the new runtime result.
- Message Events now has a per-row rerun action.
- The rerun result shows the new target Agent, intent, confidence, risk level,
  and related business objects.
- Follow-up Tasks now uses the resizable table pattern with stable widths and
  horizontal scrolling.
- Bumped local app version to `0.14.1`.

## Manual Acceptance

1. Restart the API so the new route is loaded.
2. Open `http://localhost:5173/#messages`.
3. Search a historical message such as `简历`, `群里有人问价格没人回复`, or `能处理吗`.
4. Click the rerun icon in that row and confirm.
5. Check the row refreshes with the expected Agent:
   - `多少钱/报价/方案` -> sales lead Agent.
   - `登录不了/报错` -> support ticket Agent.
   - `群里/没人回复/高意向` -> community ops Agent.
   - `简历/JD/岗位` -> recruiting HR Agent.
   - unclear messages -> manual inbox.
6. Open the related module and verify the regenerated business object appears.
