# WorkBuddy OSS v0.15.4 Feishu Approval Cards

v0.15.4 continues the v0.15.x Feishu production loop. This version turns
internal approval cards into a real audited workflow instead of a UI-only
button.

## What Changed

- Added Feishu approval card preview and send APIs:
  - `GET /api/approvals/{id}/feishu-card-preview`
  - `POST /api/approvals/{id}/feishu-card`
- Added a dedicated approval chat ID runtime setting in Config Center:
  `FEISHU_APPROVAL_CHAT_ID`.
- Added Feishu interactive card send support in the adapter layer.
- Added approval card callback handling for:
  - approve
  - reject
  - view detail
- Added idempotent callback behavior so already processed approvals are audited
  and ignored instead of silently flipping status.
- Fixed encrypted Feishu callback handling for approval cards by decrypting
  before callback classification.
- Added approval card `AgentRun=feishu_approval_card_adapter` audit records for:
  - card send
  - approve callback
  - reject callback
  - detail callback
- Added approval context fields:
  - `card_preview`
  - `card_history`
- Added `飞书卡片` action and card history display to the approvals page.
- Bumped local app/API/runtime version to `0.15.4`.

## Manual Acceptance

1. Start API and Web:
   - `.venv/bin/uvicorn apps.api.main:app --host 127.0.0.1 --port 8000`
   - `npm --prefix apps/web run dev -- --host 127.0.0.1 --port 5173`
2. Open `http://127.0.0.1:5173/#system`
3. In Config Center -> 飞书, confirm:
   - App ID / Secret are configured
   - `approval_chat_id` is filled with an internal test chat
   - send mode matches your acceptance window
4. Open `http://127.0.0.1:5173/#approvals`
5. Pick a `pending_review` approval and click `飞书卡片`.
6. Confirm the modal shows:
   - current mode (`真实发送模式` or `模拟模式`)
   - target chat ID or missing requirements
   - card JSON preview
7. If running a real acceptance window, confirm the card arrives in the Feishu
   internal approval chat.
8. Click `通过` or `拒绝` in Feishu.
9. Back in WorkBuddy, open the approval detail drawer and confirm:
   - approval status changed
   - `卡片操作历史` recorded the callback
   - callback time is shown in Beijing time
10. If Feishu callback encryption is enabled, repeat once and confirm the
    callback still lands correctly.
