# WorkBuddy OSS v1.0.0 Private Deployment Start

`v1.0.0` starts the private-deployment usable line. This first step chooses a
free deployment target and adds the production-oriented deployment package.

## What Changed

### Added

- Added `docs/PRIVATE_DEPLOYMENT.md`.
- Added `deploy/oci-free/` as the Oracle Cloud Always Free deployment target.
- Added production Dockerfiles for API and Web.
- Added a Caddy HTTPS reverse proxy config.
- Added Ubuntu bootstrap script with Docker, UFW, and fail2ban setup.
- Added deployment and remote health-check scripts.
- Added `.dockerignore` to prevent local secrets, databases, and generated
  artifacts from entering Docker build context.
- Added `scripts/check_private_deployment.py`.
- Added `npm run check:private-deploy`.

### Optimized

- The frontend now supports same-origin API deployment with `VITE_API_BASE_URL=/api`.
- Production Web is served as static files through nginx behind Caddy.
- API, Postgres, Redis, runtime jobs, and Feishu worker stay on the internal
  Docker network; only Caddy exposes `80` and `443`.

### Fixed

- The previous deployment shape still used a dev-style Vite web server. The OCI
  target now builds a static Web image.
- Docker build context now excludes local env files, databases, generated docs,
  and local video artifacts.

## Final Result

The selected free deployment target is:

```text
Oracle Cloud Always Free Ampere A1
  -> Ubuntu LTS
  -> Docker Compose
  -> Caddy HTTPS
  -> Web/API/Postgres/Redis/workers internal network
```

Actual remote deployment still requires the account owner to create the Oracle
instance, configure security rules, and provide SSH access or run the scripts.

## Manual Acceptance

Local checks:

```bash
npm run build:web
.venv/bin/python -m compileall apps/api scripts
npm run db:migrate
npm run check:private-deploy
npm run check:rc
```

Remote checks after the OCI VM exists:

```bash
sudo bash deploy/oci-free/bootstrap_ubuntu.sh
cp deploy/oci-free/.env.production.example .env.production
bash deploy/oci-free/deploy.sh
bash deploy/oci-free/check_remote.sh https://<your-domain>
```
