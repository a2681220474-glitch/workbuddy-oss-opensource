# WorkBuddy OSS v0.17.0 Team Workbench Complete

`v0.17.0` marks the end of the `v0.16.x` major version and the start of the `v0.17.x` track.

This release closes the team workbench and permissions milestone:

- local team users and current-operator switching
- role-based approval and write permissions
- user-referenced task assignees
- `GET /api/workbench/me` for my tasks, my approvals, overdue, and unassigned queue
- unified audit ledger `GET /api/audit-logs`
- audit coverage for task updates, processing records, approvals, sends, config updates, SLA updates, community task completion, lead/candidate/ticket updates
- admin-only configuration changes enforced in backend and reflected in frontend
- dashboard upgraded from system counters to a real team workbench
- dedicated audit page and assignee-aware task table

## Verification

- `npm run build:web`
- `.venv/bin/python -m compileall apps/api`
- verified `GET /api/workbench/me`
- verified `GET /api/audit-logs`
- verified non-admin config mutation returns `403`
- verified dashboard, audit page, and tasks page in the in-app browser

## Manual Acceptance

1. Open `http://127.0.0.1:5173/#dashboard`
2. Confirm `我的工作台`, `待认领任务`, and `统一操作审计` are visible
3. Open `http://127.0.0.1:5173/#team` and switch current operator
4. Open `http://127.0.0.1:5173/#tasks`, assign or update a task, then confirm it appears in `我的待办`
5. Open `http://127.0.0.1:5173/#audit` and confirm the action is recorded
6. Switch to a non-admin account and verify config save actions are blocked
