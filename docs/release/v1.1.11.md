# WorkBuddy OSS v1.1.11 飞书卡片回调诊断

`v1.1.11` 针对飞书审批卡片点击“通过/拒绝”后提示“目标回调服务当前未在线”的问题做诊断收口。结论是：飞书卡片按钮点击需要公网 HTTP 回调地址，长连接 worker 只能接收消息事件，不能接收卡片按钮回调。

## 本版做了什么

- 新增 `WORKBUDDY_PUBLIC_BASE_URL` 配置项，用于生成公网回调地址。
- 飞书诊断接口新增 `card_callback` 字段。
- 飞书诊断页新增“卡片按钮回调”区块。
- 明确显示飞书卡片按钮应配置的回调地址：`/api/channels/feishu/webhook`。
- 自动验收新增飞书审批卡片按钮回调模拟，确认 HTTP webhook 可达时能更新审批状态。
- 更新 README、release、handoff、Known Issues、Obsidian 项目记忆和版本号。

## 新增能力

- 飞书诊断页可直接解释“目标回调服务当前未在线”的含义。
- 自动验收能证明 WorkBuddy 端具备处理飞书卡片回调的能力。
- 配置 `WORKBUDDY_PUBLIC_BASE_URL=https://你的公网域名` 后，诊断页会生成完整飞书卡片回调地址。

## 优化项

- 把“飞书消息长连接在线”和“飞书卡片按钮回调在线”拆成两个概念，避免误判。
- README 更新到当前 `v1.1.x` 本地正式收口状态，不再停留在早期 Beta 叙述。
- `.env.example` 明确说明飞书审批卡片按钮也需要公网 HTTPS webhook。

## 修复项

- 修复飞书卡片回调离线问题缺少产品内诊断和文档解释的问题。
- 修复连接器自动验收没有覆盖飞书审批卡片按钮回调的问题。
- 修复 README 中配置保存路径、当前范围和版本状态过旧的问题。

## 最终结果

- 本地版本推进到 `v1.1.11`。
- WorkBuddy 代码端可处理飞书审批卡片按钮回调。
- 如果飞书端仍提示“目标回调服务当前未在线”，需要配置公网 HTTPS 地址并把飞书后台卡片交互回调指向：

```text
https://你的公网域名/api/channels/feishu/webhook
```

- 远程 ECS 已在用户后续授权后受控升级到 `v1.1.11`，HTTPS 入口和飞书卡片回调地址已生成；未触发真实外发。

## 人工验收

1. 打开 `http://localhost:5173/#feishu`。
2. 登录后确认顶部显示 `v1.1.11 飞书卡片回调诊断`。
3. 查看“卡片按钮回调”区块。
4. 如果没有公网地址，确认页面提示需要配置 `WORKBUDDY_PUBLIC_BASE_URL`。
5. 启动 HTTPS tunnel 或使用公网部署地址。
6. 设置 `WORKBUDDY_PUBLIC_BASE_URL=https://你的公网域名` 并重启 API。
7. 在飞书后台把卡片交互/机器人回调地址配置为诊断页显示的完整 URL。
8. 发送一张 WorkBuddy 审批卡片到测试群。
9. 在飞书卡片里点击“通过”或“拒绝”。
10. 回到 WorkBuddy 审批详情，确认审批状态变化，并且卡片操作历史出现回调记录。

当前阿里云 ECS 可用地址：

```text
WORKBUDDY_PUBLIC_BASE_URL=https://workbuddy.example.com
飞书卡片回调=https://workbuddy.example.com/api/channels/feishu/webhook
```
