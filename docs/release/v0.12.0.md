# WorkBuddy OSS v0.12.0 Release Candidate

v0.12.0 is the first open-source release candidate for the Business Alpha Beta loop.

## Release Goal

Make the project easy to inspect, demo and validate from a clean local setup:

- a first-run guide that points to one-click Beta validation
- release notes that explain the current product boundary
- demo reset documentation for repeatable demos
- full-page screenshots for README and video production
- a video script that can be used directly with the screenshot set
- cold-start validation from an empty SQLite database

## Product Scope

The release candidate covers:

- MessageEvent import and routing
- Support ticket and knowledge workflow
- Sales lead workflow and approval drafts
- Community operations workflow
- Recruiting and onboarding workflow
- KnowledgeGap / KnowledgeItem loop
- Report generation
- Approval queue and AgentRun audit
- Feishu diagnostics and safe mock-send defaults
- WeCom and DingTalk mock adapter boundaries

Out of scope for this release candidate:

- production permission system
- billing and tenant administration
- complex RAG / vector database
- Celery / Redis background job topology
- real WeCom or DingTalk accounts
- real external sending by default

## Human Acceptance

1. Start API and Web locally.
2. Open `http://localhost:5173/#demo`.
3. Click `一键准备 Beta 验收`.
4. Confirm the validation report shows `8/8`.
5. Review Dashboard, Messages, Business Objects, Tickets, Leads, Community, Candidates, Knowledge, Reports, Approvals and AgentRuns.
6. Confirm external replies still enter approval before any send action.
7. Confirm Feishu pages show safe mock-send mode unless real sending is intentionally configured.

## Validation Commands

```bash
npm run build:web
curl http://localhost:8000/health
```

Cold-start validation should use an empty SQLite file, then call:

```bash
curl -X POST http://localhost:8012/api/demo/prepare
```

Expected validation result:

```text
Business Alpha Beta 验收报告: 8/8
```
