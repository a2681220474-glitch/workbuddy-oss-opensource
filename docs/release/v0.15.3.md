# WorkBuddy OSS v0.15.3 Feishu Worker Observability

v0.15.3 continues the v0.15.x Feishu production loop. This version focuses on
making the Feishu worker diagnosable while it is running, idle, stale, failed,
or recovering.

## What Changed

- Added continuous Feishu stream worker heartbeat writes.
- Added heartbeat count, last heartbeat time, last success time, and last
  failure time to the worker status file.
- Added recent processed event history to the worker status file.
- Added recent structured error history with event type, payload hash, recovery
  hint, and error text.
- Preserved replayable raw payloads when stream event processing fails.
- Updated worker `AgentRun` prompt version to
  `v0.15.3-stream-observability-v1`.
- Added a `Worker 状态与恢复` section to the Feishu diagnostics page.
- Bumped local app/API/runtime version to `0.15.3`.

## Manual Acceptance

1. Start API and Web:
   - `.venv/bin/uvicorn apps.api.main:app --host 127.0.0.1 --port 8000`
   - `npm --prefix apps/web run dev -- --host 127.0.0.1 --port 5173`
2. Run `npm run check:feishu-stream` and confirm configuration is valid.
3. Start the real worker only during a Feishu acceptance window:
   - `npm run dev:feishu-stream`
4. Open `http://localhost:5173/#feishu`.
5. Confirm the worker section shows:
   - status
   - process state
   - heartbeat time and count
   - recent success/failure time
   - recovery commands
   - recent processed events or recent errors
6. Send a test message from Feishu and confirm recent processed events update.
7. Stop the worker and confirm diagnostics eventually mark the worker as offline
   or stale with clear recovery steps.
