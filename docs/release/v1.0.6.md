# WorkBuddy OSS v1.0.6 RBAC Boundary

`v1.0.6` adds a centralized role boundary for write actions.

## What Changed

- Added an API RBAC middleware for unsafe methods.
- Admins can write across all private-management surfaces.
- Approvers can perform approval and business-record actions.
- Handlers can update operational workflow objects.
- Read-only users are blocked from unsafe writes.
- Non-admin web navigation hides the most sensitive system entries.

## Optimized

- The permission boundary is now enforced before requests enter business
  routers, while existing route-level checks remain in place.

## Fixed

- Fixed the gap where some write endpoints depended only on UI affordances or
  individual route checks.

## Final Result

RBAC is now enforced as a shared backend boundary for local-first private use.

## Manual Acceptance

1. Log in as an admin and confirm system configuration still saves.
2. Log in as a non-admin user.
3. Confirm sensitive system entries are no longer prominent in the sidebar.
4. Attempt an admin-only write and confirm the API returns `403`.
