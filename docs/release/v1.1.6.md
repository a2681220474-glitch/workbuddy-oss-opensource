# WorkBuddy OSS v1.1.6 发布前综合差距审计

`v1.1.6` 将九项后续基线、停止开发标准和部署边界变成可执行、可展示、可自动回归的发布前总审计。

## 本版做了什么

- 新增统一发布差距审计服务。
- 在配置中心展示九项基线和两阶段停止开发结论。
- 修复发布候选脚本无法识别 `v1.1.x` 的问题。
- 新增隔离式发布差距验收脚本。

## 新增能力

- 配置中心新增“发布前综合差距审计”。
- 九项基线统一分为：已完成、需人工授权、需部署环境、本地缺口。
- 明确展示“本地代码已就绪”与“正式私有化可投入”是两个不同结论。
- 阶段一停止线用于判断是否停止继续堆大功能。
- 阶段二停止线明确要求真实团队连续运行两周、无 P0/P1 且用户确认。
- `npm run check:release-gap-audit` 提供 9 项隔离检查。

## 优化

- 发布前结论不再散落在路线图、Known Issues 和部署文档中。
- 历史连接器证据只作为历史样本，不会被标成新的真实复验。
- Docker/Postgres/Redis 未实跑时明确显示“需部署环境”，不再被静态 Compose 检查掩盖。
- 远程 ECS 冻结状态和版本直接显示在审计结果中。

## 修复

- 修复 `npm run check:rc` 只支持 `v0.20.x` 和 `v1.0.0`、在 `v1.1.x` 错误失败的问题。
- 修复停止开发标准只能人工阅读、无法自动判断的问题。
- 修复“本地代码完成”容易被误解为“已经正式投入使用”的表达风险。
- 修复真实团队两周运行观察没有独立状态的问题。

## 最终结果

- 九项基线当前结论：6 项已完成、1 项需人工授权、2 项需部署环境、本地代码缺口 0 项。
- 本地代码与自动验收已达到停止继续堆大功能的条件。
- 正式私有化投入仍需连接器人工复验和 Docker/Postgres/Redis 部署验收。
- 停止当前产品线主动开发仍需真实团队连续运行两周并由用户确认。
- 远程 ECS 继续冻结在 `v1.0.1`，本版未部署远程，也未触发真实外发。
- 隔离 Chrome 验收确认审计卡片完整显示 9 项基线，页面无横向溢出。

## 人工验收

1. 打开 `http://localhost:5173/#config`。
2. 找到“发布前综合差距审计”。
3. 确认显示：已完成 6、需人工 1、需部署环境 2、本地缺口 0。
4. 确认“停止大功能开发”显示本地代码已就绪，但仍需部署和人工验证。
5. 确认“停止当前产品线主动开发”仍要求真实团队运行观察。
6. 运行 `npm run check:release-gap-audit`，确认 9 项通过。
7. 运行 `npm run check:rc`，确认 `v1.1.6` 发布候选检查通过。

## 自动验证

```bash
npm run check:release-gap-audit
npm run check:rc
npm run check:product-workflow
npm run check:connector-acceptance
npm run check:rag-workflow
npm run check:local-acceptance
npm run build:web
npm run check:web-bundle
npm run check:release
.venv/bin/python -m compileall apps/api scripts
```

隔离浏览器验收使用临时数据库和禁止外发策略完成，不修改正式业务数据。
