# WorkBuddy OSS v0.19.5 Knowledge Document Import

`v0.19.5` continues the `v0.19.x` knowledge and RAG formalization line. This
release adds a manual-preview import workflow for Markdown, FAQ text, and CSV
knowledge files.

## What Changed

### Added

- Added `POST /api/imports/knowledge/preview`.
- Added `POST /api/imports/knowledge/confirm`.
- Added Markdown parsing by heading sections.
- Added FAQ text parsing for `Q/A`, `Question/Answer`, and `问/答` style files.
- Added CSV parsing for knowledge columns such as title, question, answer,
  category, mode, and status.
- Added a knowledge import area to the Import page.
- Added import preview table before confirmation.
- Confirmation creates KnowledgeItem or KnowledgeGap records and writes an
  ImportBatch with source filename metadata.
- Imported KnowledgeItem records write version snapshots.
- Confirmed imports write audit logs.

### Optimized

- Message import and knowledge import now live on the same import page but use
  separate forms and preview tables.
- Knowledge import supports default category, default item/gap mode, and draft
  versus published import state.

### Fixed

- Teams no longer need to manually copy FAQ entries one by one into the
  knowledge table.
- Source file information is preserved in ImportBatch metadata for later audit
  and troubleshooting.

## Final Result

Knowledge can now enter WorkBuddy through a safe two-step workflow:

```text
Markdown / FAQ / CSV
  -> preview parsed KnowledgeItem / KnowledgeGap rows
  -> human confirms
  -> ImportBatch + KnowledgeItem / KnowledgeGap
  -> version snapshot + audit log
```

This release does not add automatic attachment parsing, semantic chunking, or
vector indexing. Those remain later `v0.19.x` work.

## Manual Acceptance

Run:

```bash
npm run build:web
.venv/bin/python -m compileall apps/api
npm run db:migrate
```

Then open `http://127.0.0.1:5173/#import` and verify:

1. The page still supports normal message import.
2. The `知识库导入` area accepts Markdown, FAQ text, and CSV content.
3. Clicking `生成知识导入预览` fills the preview table.
4. Rows show type, title/question, category, status, answer, and warnings.
5. Clicking `确认导入` creates KnowledgeItem and/or KnowledgeGap records.
6. Imported KnowledgeItem records appear under `http://127.0.0.1:5173/#knowledge`.

API smoke test:

```bash
.venv/bin/python -c "from fastapi.testclient import TestClient; from apps.api.main import app; c=TestClient(app); print(c.post('/api/imports/knowledge/preview', json={'source_type':'faq','filename':'faq.txt','content':'Q: 如何重置密码？\\nA: 在设置页点击重置密码。','default_category':'support','default_mode':'item','publish':False}).status_code)"
```
