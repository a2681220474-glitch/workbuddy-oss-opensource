# WorkBuddy OSS v0.19.4 Knowledge Quality Governance

`v0.19.4` continues the `v0.19.x` knowledge and RAG formalization line. This
release adds knowledge expiry reminders and a quality governance panel so the
team can decide which knowledge should be reviewed, optimized, or archived.

## What Changed

### Added

- Added knowledge item quality fields:
  - `review_due_at`
  - `last_reviewed_at`
  - `quality_status`
  - `quality_score`
- Added Alembic migration `20260605_0004_knowledge_quality.py`.
- Added `GET /api/knowledge/quality`.
- Added the knowledge page `质量治理` tab.
- Added quality lists for expired items, review-due items, optimization
  candidates, archive suggestions, and repeated knowledge gaps.
- Added one-click `已复审` action that refreshes the review date and restores a
  healthy quality status.

### Optimized

- KnowledgeItem list now shows quality status, quality score, and review due
  date.
- KnowledgeItem detail drawer now shows review and quality fields.
- Status colors and labels now cover knowledge quality states.

### Fixed

- Knowledge governance can now identify stale published items that have never
  been hit.
- Repeated pending KnowledgeGap records are now surfaced in a quality panel
  instead of being hidden only in the raw gap table.

## Final Result

Knowledge maintenance now has a basic quality loop:

```text
KnowledgeItem
  -> review due / expired reminder
  -> hit score quality signal
  -> optimize or archive suggestion
  -> human review action
  -> version, processing record, audit log
```

This remains a lightweight relational implementation. It does not add vector
retrieval, automated semantic quality scoring, or a graph database.

## Manual Acceptance

Run:

```bash
npm run build:web
.venv/bin/python -m compileall apps/api
npm run db:migrate
```

Then open `http://127.0.0.1:5173/#knowledge` and verify:

1. The `质量治理` tab is visible.
2. The summary shows total items, published items, expired count, review-due
   count, optimization candidates, archive suggestions, pending gaps, and
   repeated gaps.
3. KnowledgeItem rows show quality status, quality score, and review due date.
4. Editing a KnowledgeItem can save review date, quality status, and quality
   score with a change summary.
5. Clicking `已复审` updates the item to healthy quality status and sets a future
   review date.
6. KnowledgeItem details show quality fields.

API smoke test:

```bash
.venv/bin/python -c "from fastapi.testclient import TestClient; from apps.api.main import app; c=TestClient(app); print(c.get('/api/knowledge/quality').status_code)"
```
