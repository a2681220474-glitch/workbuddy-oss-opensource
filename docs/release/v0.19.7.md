# WorkBuddy OSS v0.19.7 Agent Knowledge Citations

`v0.19.7` completes the `v0.19.x` knowledge and RAG formalization line by
connecting lightweight retrieval to Agent-generated approval drafts.

## What Changed

### Added

- Agent message routing now retrieves top published KnowledgeItem references
  before creating the AgentRun.
- AgentRun `model_output_json` records `knowledge_references` with item id,
  title, category, score, reasons, snippet, and hit id.
- Approval context now returns the AgentRun knowledge references.
- Approval detail now displays a `知识引用` section below the AI draft.

### Optimized

- Knowledge citations are shown to human reviewers without being inserted into
  the outbound draft content.
- Retrieval uses the same scoring, snippet, and KnowledgeHit recording helpers
  as the knowledge search API, keeping manual search and Agent citation behavior
  aligned.
- Reviewers can see why a draft referenced a knowledge item before approving,
  editing, or rejecting it.

### Fixed

- Agent drafts no longer lose source traceability after the lightweight
  retrieval layer finds relevant published knowledge.
- Approval review now exposes the knowledge basis for AI suggestions instead of
  leaving it hidden in backend-only routing context.

## Final Result

WorkBuddy now has a complete v0.19 knowledge loop:

```text
published KnowledgeItem
  -> lightweight retrieval
  -> KnowledgeHit
  -> AgentRun knowledge_references
  -> approval detail citations
  -> human review before send
```

This is still lightweight retrieval, not vector RAG. It is the final bridge
needed before `v0.20.x` packaging and open-source release preparation.

## Manual Acceptance

Run:

```bash
npm run build:web
.venv/bin/python -m compileall apps/api
npm run db:migrate
```

Then verify in the UI:

1. Make sure at least one KnowledgeItem is `published`.
2. Send or import a message containing words from that knowledge item.
3. Open the generated approval detail.
4. Confirm the AI draft still reads normally.
5. Confirm the new `知识引用` section shows the referenced KnowledgeItem, score,
   reasons, snippet, and hit id.
6. Approve/edit/reject as usual and confirm the citation is only reviewer
   context, not forced into the outbound reply.

API smoke check:

```bash
.venv/bin/python -c "from fastapi.testclient import TestClient; from apps.api.main import app; c=TestClient(app); print(c.get('/health').json()['version'])"
```

Expected version: `0.19.7`.
