# WorkBuddy OSS v1.1.5 连接器安全复验

`v1.1.5` 完成飞书与企业微信连接器的无外发风险自动复验，并为真实测试发送增加强制人工授权门禁。

## 本版做了什么

- 新增飞书/企微统一的发布前安全复验摘要。
- 新增隔离式连接器验收脚本，使用临时数据库和虚拟凭证运行。
- 统一检查接收链路、业务路由、Mock 发送、失败回放和发送退避策略。
- 在两个诊断页增加“发布前安全复验”面板。

## 新增能力

- `npm run check:connector-acceptance` 一次完成 9 项连接器安全检查。
- 诊断接口新增 `safe_acceptance`，明确区分自动安全复验与真实外发证据。
- 真实测试发送要求 `confirm_real_send=true`、精确授权短语和全局外发开关三项同时满足。
- 飞书失败接收事件可在隔离验收中验证回放入库。
- 本地飞书/企微验收脚本支持通过 `X-WorkBuddy-User` 读取登录后的诊断接口。

## 优化

- 飞书和企微共用相同的验收判定与真实外发授权规则。
- 诊断页直接展示配置、接收、业务链路、Mock 发送和失败恢复五类检查。
- 真实发送弹窗关闭或发送成功后自动清空授权短语，避免误复用。
- 已有真实发送记录只作为证据展示，不会被自动验收脚本再次触发。

## 修复

- 修复真实测试发送只依赖布尔确认，误操作防护不足的问题。
- 修复开启全局外发开关后诊断页真实发送按钮缺少二次强确认的问题。
- 修复本地连接器验收脚本在正式登录门禁启用后无法读取诊断接口的问题。
- 修复连接器最终复验缺少独立、可重复、不会污染本地数据的自动检查入口。

## 最终结果

- 9 项连接器隔离验收全部通过。
- 自动验收不会连接飞书或企微外部发送接口，不会读取真实密钥，不会修改真实本地数据库。
- 历史真实接收与真实发送样本继续保留，但本版没有自动发送任何真实消息。
- 真实外发必须由用户确认目标和内容后手工输入 `CONFIRM WORKBUDDY REAL SEND`。
- 远程 ECS 继续冻结在 `v1.0.1`，本版未部署远程。

## 人工验收

1. 打开 `http://localhost:5173/#feishu` 和 `http://localhost:5173/#wecom`。
2. 确认“发布前安全复验”展示配置、接收、业务链路、Mock 发送和失败恢复状态。
3. 点击“安全模拟发送”，确认提示不会触达外部平台，刷新后 Mock 检查保持通过。
4. 打开“真实测试发送”，不输入授权短语，确认按钮不可点击。
5. 输入错误短语，确认按钮仍不可点击；关闭再打开弹窗，确认短语被清空。
6. 运行 `npm run check:connector-acceptance`，确认 9 项全部通过。
7. 只有需要新的真实平台验收时，才确认测试目标、测试内容和全局外发开关，并手工输入授权短语。
