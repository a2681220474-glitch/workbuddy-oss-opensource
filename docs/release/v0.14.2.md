# WorkBuddy OSS v0.14.2 Agent Trace Detail

v0.14.2 makes Agent runs easier to inspect from the product UI. The goal is to
help local testing answer: why did this message route here, did it call a real
model, what failed, and which business actions were generated?

## Changes

- Added `GET /api/agent-runs/{run_id}` for direct run lookup.
- Added a reusable AgentRun detail drawer.
- Message Events can now open the latest AgentRun for each message.
- Run details show route reason, target Agent, intent, confidence, model
  provider/model, token usage, latency, LLM request payload, model output,
  runtime error/fallback reason, and generated actions.
- Reports now use the resizable table pattern with stable column widths and
  horizontal scrolling.
- Bumped local app version to `0.14.2`.

## Manual Acceptance

1. Restart the API and refresh the Web app.
2. Open `http://localhost:5173/#messages`.
3. Search a real message and click the eye icon in the operation column.
4. Confirm the drawer shows:
   - target Agent and intent,
   - route reason,
   - model provider/model,
   - LLM request and model output JSON,
   - generated actions.
5. Open `http://localhost:5173/#runs` and click `查看` on a row.
6. Confirm the same detail drawer appears from the run log page.
7. Open `http://localhost:5173/#reports` and verify the table can scroll and
   column widths can be dragged from the header edge.
