# WorkBuddy OSS v0.19.1 Knowledge Detail And Versions

`v0.19.1` starts the `v0.19.x` knowledge and RAG formalization line. This
version turns knowledge from list-only records into inspectable objects with
source references and version snapshots.

## What Landed

- Added `knowledge_item_versions` table.
- Added Alembic migration `20260604_0002_knowledge_versions`.
- SQLite local startup now creates missing tables safely.
- Added knowledge detail APIs:
  - `GET /api/knowledge/gaps/{gap_id}`
  - `GET /api/knowledge/items/{item_id}`
- Knowledge gap detail now includes:
  - source references
  - related knowledge items
  - processing records
  - timeline
- Knowledge item detail now includes:
  - source gap
  - source references
  - version history
  - processing records
  - timeline
- Accepting a gap creates an initial knowledge item version.
- Updating a knowledge item creates a new version snapshot.
- Creating knowledge from a ticket writes an initial version snapshot.
- Knowledge page detail drawer now loads the formal detail API and shows source
  references, versions, related items, and timeline.

## Verification

- `npm run db:migrate`
- `npm run build:web`
- `.venv/bin/python -m compileall apps/api`
- `curl http://127.0.0.1:8000/api/knowledge/items/<id>`
- `curl http://127.0.0.1:8000/api/knowledge/gaps/<id>`

## Manual Acceptance

1. Open `http://127.0.0.1:5173/#knowledge`.
2. Open a KnowledgeGap detail drawer.
3. Confirm source references and timeline are visible.
4. Open a KnowledgeItem detail drawer.
5. Confirm source references and version history are visible.
6. Update a KnowledgeItem status or content.
7. Reopen detail and confirm a new version appears.
