# WorkBuddy OSS v1.1.15 真实验收收口

`v1.1.15` 将用户于 2026-06-15 确认的飞书真实人工验收结果写入发布审计，不重复修改已经通过的审批卡片实现。

## 本版做了什么

- 将 `v1.1.14` 飞书真实人工验收标记为完成。
- 将远程 ECS `v1.1.14` 受控升级标记为完成。
- 把九项发布基线更新为 8 项完成、0 项人工待验、1 项部署待验。
- 将剩余部署事项收敛为 PostgreSQL 备份恢复隔离演练。

## 新增

- 发布审计记录人工验收日期、验收版本和四项验收内容。
- 发布审计记录远程 ECS 当前已验收版本。
- 自动检查防止已完成的真实验收和远程升级再次被误报为待办。

## 优化

- 配置中心由“远程 ECS 冻结版本”改为“远程 ECS 当前已验收版本”。
- 阶段一结论会准确提示只剩部署恢复演练。
- 后续远程升级仍保持单独授权和部署前备份边界。

## 修复

- 修复发布审计长期停留在 6 项完成、1 项人工待验、2 项部署待验的过时状态。
- 修复配置中心仍显示远程 ECS 冻结在 `v1.0.1` 的过时信息。
- 修复交接和 Known Issues 仍要求重复执行飞书卡片人工验收的问题。

## 最终结果

- 当前版本：`v1.1.15`。
- 本地正式收口状态保持通过。
- 飞书新消息只入库一次：通过。
- 全系统显示北京时间：通过。
- 审批后原卡片只保留“已通过/已拒绝 + 查看详情”：通过。
- 查看详情不再报 `200672`：通过。
- 本地代码缺口：0。
- 九项基线：8 项完成、0 项人工待验、1 项部署待验。
- 远程 ECS 当前仍运行 `v1.1.14`；本版不自动部署远程。
- 真实外发仍需用户明确授权。
- 远程 ECS 后续不随每个小版本自动升级。

## 人工验收

1. 打开配置中心“发布前综合差距审计”。
2. 确认显示已完成 8、需人工 0、需部署环境 1、本地缺口 0。
3. 确认真实连接器最终验收显示“已完成”。
4. 确认远程 ECS 当前已验收版本显示 `v1.1.14`。
