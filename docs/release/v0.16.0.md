# WorkBuddy OSS v0.16.0 Team Workbench Foundations

`v0.16.0` starts the team workbench and permissions major version.

## What Changed

- Added local user model `LocalUser`.
- API now seeds a default admin:
  - username: `local_admin`
  - display name: `本地管理员`
- Added current-user endpoint:
  - `GET /api/auth/me`
- Added local user management endpoints:
  - `GET /api/users`
  - `POST /api/users`
  - `PATCH /api/users/{id}`
- Frontend now has a `团队成员` page.
- Sidebar now supports switching the current local operator.
- Approval actions are now restricted to:
  - `admin`
  - `approver`
- Business-object processing records now write the current operator identity
  instead of the fixed `本地账号`.
- Read-only users are blocked from writing processing records.

## Manual Acceptance

1. Start API and Web.
2. Open `http://127.0.0.1:5173/#team`.
3. Confirm the default `local_admin` user exists.
4. Create:
   - one `approver`
   - one `handler`
   - one `readonly`
5. Switch the current user from the sidebar account area.
6. Open `http://127.0.0.1:5173/#approvals`:
   - `admin` or `approver` can approve/send
   - `handler` and `readonly` are blocked
7. Open any business object detail drawer and add a processing record as
   `handler`.
8. Confirm the record shows the chosen operator name.
