# WorkBuddy OSS v0.19.3 Knowledge Workflow And Graph

`v0.19.3` continues the `v0.19.x` knowledge and RAG formalization line. This
release makes knowledge maintenance more explicit and adds an graph-style
graph view without introducing a graph database.

## What Changed

- Added the formal knowledge item status flow:
  `draft -> pending_review -> published -> archived`.
- Knowledge item edits now accept title, answer, category, status, and a change
  summary.
- Knowledge item edit, review, publish, archive, gap accept, and gap ignore
  actions now write processing records and audit logs.
- Knowledge item updates continue to write version snapshots with more specific
  change types such as `submit_review`, `publish`, and `archive`.
- Added `GET /api/knowledge/graph` to build a lightweight knowledge graph from
  existing KnowledgeItem, KnowledgeGap, SourceMessage, Ticket, AgentRun,
  Category, Hit, and KnowledgeVersion records.
- Added the `知识图谱` tab to the knowledge page with clickable SVG graph nodes.
- Version labels updated to `v0.19.3`.

## Graph Scope

The graph is built from existing relational data and includes these edge types:

- `source_of`
- `created_from`
- `hit_by`
- `same_category`
- `version_of`

This release intentionally does not add Neo4j, vector search, cross-tenant graph
reasoning, or full RAG retrieval.

## Verification

Run:

```bash
npm run build:web
.venv/bin/python -m compileall apps/api
npm run db:migrate
```

Then open `http://127.0.0.1:5173/#knowledge` and verify:

1. KnowledgeItem rows can be edited with a change summary.
2. Draft items can be submitted for review.
3. Pending review items can be published.
4. Items can be archived.
5. The `知识图谱` tab shows nodes and relationship lines.
6. Clicking KnowledgeItem or KnowledgeGap nodes opens details; clicking message,
   ticket, or AgentRun nodes jumps to the source page.
