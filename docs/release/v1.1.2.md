# WorkBuddy OSS v1.1.2 前端按需加载与拆包

`v1.1.2` 完成九项后续基线中的第 8 项：页面级懒加载和前端构建产物拆分。

## 本版做了什么

- 将登录页和 22 个工作台业务页面改为 `React.lazy` 动态导入。
- 保留现有 Hash 导航、角色菜单、登录状态和页面参数行为。
- 页面加载期间显示稳定的居中加载状态。
- 将 React、Ant Design 图标和日期依赖拆为稳定 vendor chunk。
- 新增 `npm run check:web-bundle` 构建产物预算检查。

## 新增能力

- 首次打开只加载应用外壳和当前所需页面。
- 切换到消息、审批、知识、配置、诊断等页面时才下载对应模块。
- 生产构建自动生成独立页面 chunk，浏览器可以长期缓存公共依赖。
- 自动检查最大 chunk、入口 chunk、懒加载页面数量和 vendor 分组。

## 优化

- 最大 JavaScript chunk 从约 `1,540.40 kB` 降至 `495.01 kB`，减少约 67.9%。
- 构建产物从单一主包拆为 48 个 JavaScript chunk。
- 23 个页面 chunk 独立输出，单个页面主体约 `2.46-36.77 kB`。
- Vite 构建不再出现超过 500 kB 的 chunk 警告。

## 修复

- 修复访问登录页或工作台任意页面时提前打包全部业务页面的问题。
- 避免粗粒度 Ant Design vendor 分组造成循环 chunk。
- 用自动预算检查防止后续静态导入导致主包体积回退。

## 最终结果

- 页面导航、鉴权和角色控制保持原行为。
- `npm run build:web` 无大包警告。
- `npm run check:web-bundle` 检查全部通过。
- 本地 API、真实数据和运行密钥未被拆包改动。
- 远程 ECS 继续冻结在 `v1.0.1`，本版未部署远程。

## 人工验收

1. 打开 `http://localhost:5173/`，确认登录页正常显示。
2. 登录后确认顶部版本显示 `v1.1.2 按需加载`。
3. 依次打开工作台、消息事件、审批队列、知识库和配置中心。
4. 确认首次进入页面时短暂显示加载状态，随后页面内容正常。
5. 刷新任意带 Hash 的页面，确认仍停留在对应功能页。
6. 打开浏览器开发者工具的 Network，确认页面模块在首次访问时按需请求。
