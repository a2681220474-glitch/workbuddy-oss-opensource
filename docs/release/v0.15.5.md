# WorkBuddy OSS v0.15.5 Feishu Acceptance Closure

`v0.15.5` closes the remaining formal gaps of the `v0.15.x` Feishu production
loop.

## What Changed

- Added traceable non-text Feishu message coverage for:
  - image
  - file
  - rich text / post
- Message events now expose:
  - `message_type_label`
  - `traceable_non_text`
  - `non_text_summary`
  - `message_tracking`
- Messages page now supports:
  - message type filtering
  - message type column
  - non-text trace detail drawer
- Adapter Test now includes Feishu sample imports for:
  - file
  - post
  - image
- Feishu diagnostics now includes:
  - `acceptance_traces`
  - `acceptance_summary`
- Added acceptance CLI script:
  - `scripts/run_feishu_acceptance.py`
  - `npm run check:feishu-acceptance`

## Manual Acceptance

1. Start API and Web.
2. Open `http://127.0.0.1:5173/#adapter-test`.
3. Import a Feishu file or rich-text sample.
4. Open `http://127.0.0.1:5173/#messages`.
5. Confirm the row shows a message type tag and `查看追踪` drawer content.
6. Open `http://127.0.0.1:5173/#feishu`.
7. Confirm `acceptance traces` show message -> object -> approval -> timeline
   coverage.
8. Run `npm run check:feishu-acceptance` and confirm at least one trace reaches
   `complete`.
