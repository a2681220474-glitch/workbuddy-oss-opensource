# WorkBuddy OSS v1.0.4 Login Bootstrap Validation UX

`v1.0.4` fixes the first-run login initialization experience when the password
does not meet the backend validation rule.

## What Changed

### Added

- Added frontend validation that requires at least 8 characters when setting
  the first local admin password.
- Added readable parsing for FastAPI `422` validation errors.

### Optimized

- The bootstrap form now explains the first password length requirement before
  submitting.
- API validation errors now show field-specific Chinese messages where possible.

### Fixed

- Fixed the confusing `Unprocessable Content` toast shown when a short password
  was submitted during first-run initialization.

## Final Result

When the first local admin password is shorter than 8 characters, the form now
shows a local validation message instead of sending the request and surfacing a
generic HTTP error.

`/health` should now report version `1.0.4`.

## Manual Acceptance

1. Open `http://localhost:5173/`.
2. On the first-run initialization screen, enter a 6-character password.
3. Click `初始化并登录`.
4. Confirm the form shows `密码至少需要 8 位`.
5. Enter an 8+ character password and confirm initialization succeeds.
