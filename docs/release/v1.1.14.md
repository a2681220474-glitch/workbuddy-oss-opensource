# WorkBuddy OSS v1.1.14 飞书一次性审批卡片

`v1.1.14` 修复飞书审批成功后原卡片仍保留“通过/拒绝”按钮的问题。

## 本版做了什么

- 按飞书 SDK `P2CardActionTriggerResponse` 模型返回卡片动作响应。
- 响应包含 `toast` 和 `card.type=raw`。
- 更新后的完整卡片放入 `card.data`。
- 已处理审批再次点击时仍返回完成态卡片，可修复此前未更新的旧卡片。

## 新增

- 自动验收校验 `toast + card.type + card.data` 完整协议结构。
- 通过和拒绝都校验完成后只剩状态按钮和“查看详情”。

## 优化

- 审批卡片真正成为一次性操作卡片。
- 幂等重复点击不会改写审批结果，但会返回正确完成态卡片。

## 修复

- 修复 `v1.1.13` 裸卡片响应不触发飞书原卡片替换。
- 修复审批已成功但“通过/拒绝”仍可见的问题。

## 最终结果

- 当前版本：`v1.1.14`。
- 点击通过后只保留“已通过 + 查看详情”。
- 点击拒绝后只保留“已拒绝 + 查看详情”。
- 消息幂等、北京时间和详情链接保持不变。
- 本地正式收口状态保持不变。
- 真实外发仍需用户明确授权。
- 远程 ECS 后续不随每个小版本自动升级。
- 远程 ECS 已受控升级到 `v1.1.14`，公网健康和飞书 worker 长连接正常。
- 部署前 PostgreSQL 备份：`/path/to/backups/workbuddy-before-upgrade-YYYYMMDD-HHMMSS.sql`。

## 人工验收

1. 点击审批 #12 当前仍保留的任一按钮，确认卡片刷新为既有审批结果和“查看详情”。
2. 发送两条全新消息，分别生成通过和拒绝审批。
3. 确认两张卡片点击后不报错，且操作按钮永久消失。
4. 点击“查看详情”，确认正常定位线上审批。
