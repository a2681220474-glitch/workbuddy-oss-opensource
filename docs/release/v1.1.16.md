# WorkBuddy OSS v1.1.16 部署恢复收口

`v1.1.16` 在远程 ECS 上完成 PostgreSQL 非破坏性隔离恢复演练，正式数据库和在线服务未中断。

## 本版做了什么

- 新增 PostgreSQL 隔离恢复演练脚本。
- 使用 `v1.1.14` 部署前备份真实恢复到临时数据库。
- 校验恢复后的表、消息、审批和 Alembic 版本。
- 自动删除临时数据库并复核正式数据库记录数未变化。
- 将九项发布基线更新为 9 项全部完成。

## 新增

- `deploy/oci-free/postgres_restore_drill.sh`。
- `npm run check:postgres-restore-drill`。
- 不含密钥的演练证据 `docs/ops/postgres_restore_drill_example.json`。
- `check:formal-release` 自动检查恢复脚本安全边界和真实演练证据。

## 优化

- 临时数据库必须使用 `workbuddy_restore_drill_` 前缀。
- 正式库和 PostgreSQL 系统库受到硬拒绝保护。
- SQL 和 custom dump 恢复遇到首个错误即终止。
- 成功或失败都会触发临时数据库清理。
- 输出记录备份大小和 SHA-256，但不记录密码。

## 修复

- 修复 PostgreSQL 只有恢复说明、没有可重复执行安全演练工具的问题。
- 修复 `.sql` 备份只能检查文件大小、无法证明真正可恢复的问题。
- 修复发布审计仍保留最后 1 项部署待验的问题。

## 最终结果

- 当前版本：`v1.1.16`。
- 九项基线：9 项完成、0 项人工待验、0 项部署待验、本地缺口 0。
- 正式私有化阶段一状态：可投入。

2. 确认显示已完成 9、需人工 0、需部署环境 0、本地缺口 0。
3. 确认“正式私有化”显示可投入。
4. 查看 `docs/ops/postgres_restore_drill_example.json`，确认临时库已删除。
