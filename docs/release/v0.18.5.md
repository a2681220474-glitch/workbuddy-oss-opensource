# WorkBuddy OSS v0.18.5 Scheduled Reports And v0.18 Closeout

`v0.18.5` completes the remaining runtime-jobs gap by making daily report
generation executable from the background worker.

## What Landed

- Runtime jobs now generate scheduled daily reports.
- Scheduled report generation covers:
  - operations daily
  - support daily
  - sales daily
  - community daily
  - recruiting progress
  - knowledge gap report
- Scheduled reports are deduplicated by tenant, report type, and Beijing date.
- Each scheduled report is committed independently so one failure does not block
  the rest of the batch.
- Runtime job cycle summaries now include generated, skipped, and failed report
  counts.
- `v0.18.x` is now code-complete for local runtime operations.

## Verification

- `ENABLE_BACKGROUND_JOBS=true .venv/bin/python -m apps.api.workers.runtime_jobs --once`
- `npm run check:background-jobs`
- `npm run check:runtime-stack`
- `npm run check:deployment`
- `npm run build:web`
- `.venv/bin/python -m compileall apps/api`

## Manual Acceptance

1. Run `ENABLE_BACKGROUND_JOBS=true .venv/bin/python -m apps.api.workers.runtime_jobs --once`.
2. Open `http://127.0.0.1:5173/#reports`.
3. Confirm scheduled reports exist for today's Beijing date.
4. Run the worker once again.
5. Confirm reports are skipped instead of duplicated.
