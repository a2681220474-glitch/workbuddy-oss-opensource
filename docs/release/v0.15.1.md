# WorkBuddy OSS v0.15.1 Feishu Encrypted Callback

v0.15.1 continues the v0.15.x Feishu production loop. This patch removes the
largest receive-side blocker from v0.15.0: encrypted Feishu HTTP callbacks are
now supported when `FEISHU_ENCRYPT_KEY` is configured.

## Changes

- Added encrypted Feishu HTTP callback decryption for payloads containing
  `encrypt`.
- Feishu callback decryption uses the official `lark-oapi` AES decryptor.
- Feishu stream worker now passes `FEISHU_ENCRYPT_KEY` into the official SDK
  event dispatcher instead of refusing to start.
- Feishu webhook parse failures are now recorded as failed `ChannelEvent`
  records with error, advice, encrypted flag, and raw payload.
- Feishu diagnostics now reports encrypted callback support as available.
- Config Center capabilities now mark Feishu encrypted callbacks as supported.
- Bumped local app/API/runtime version to `0.15.1`.

## Manual Acceptance

1. Restart the API and refresh the Web app.
2. Open `http://127.0.0.1:5173/#config` and confirm the Feishu Encrypt Key field
   can be configured from the page.
3. Open `http://127.0.0.1:5173/#feishu` and confirm the production notes mention
   encrypted callback support.
4. Run `npm run check:feishu-stream`.
5. If the Feishu app has callback encryption enabled, start
   `npm run dev:feishu-stream` and send a real Feishu test message.
6. Confirm the message enters WorkBuddy as a `MessageEvent`.
7. If decryption fails, open channel events and confirm a failed
   `feishu.webhook.parse.failed` event is recorded with an actionable error.
