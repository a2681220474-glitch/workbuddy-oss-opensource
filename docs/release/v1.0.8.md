# WorkBuddy OSS v1.0.8 Local Acceptance Script

`v1.0.8` adds a non-destructive local acceptance command.

## What Changed

- Added `scripts/check_local_acceptance.py`.
- Added `npm run check:local-acceptance`.
- The check verifies local static files, release records, API health,
  database connectivity, and auth bootstrap status.

## Optimized

- Local acceptance no longer depends only on manual clicking.
- The check avoids mutating local business data.

## Fixed

- Fixed the lack of a single local command for quick private-use readiness.

## Final Result

Local-first verification has a dedicated script that can be run before deciding
whether to deploy a later build remotely.

## Manual Acceptance

1. Start the API and Web locally.
2. Run `npm run check:local-acceptance`.
3. Confirm all checks pass.

## Verification

- `npm run check:local-acceptance`
