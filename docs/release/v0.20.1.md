# WorkBuddy OSS v0.20.1 Connector Setup Manuals

`v0.20.1` continues release-candidate preparation by turning Feishu and WeCom
real-connector setup into explicit manual workflows.

## What Changed

### Added

- Added `docs/FEISHU_SETUP.md`.
- Added `docs/WECOM_SETUP.md`.
- Extended `npm run check:release` to require both connector setup manuals.

### Optimized

- Feishu setup now separates app creation, WorkBuddy config, stream worker
  startup, receive acceptance, and send acceptance.
- WeCom setup now separates app credentials, public HTTPS callback exposure,
  URL verification, receive acceptance, trusted IP checks, and send acceptance.
- Both guides keep `ENABLE_EXTERNAL_SEND=false` as the first-pass default.

### Fixed

- Connector setup is no longer buried across older phase notes and runtime
  config documentation.
- Manual acceptance now has one Feishu checklist and one WeCom checklist.

## Final Result

New testers can now follow a connector-specific guide:

```text
Feishu app / WeCom app
  -> WorkBuddy config
  -> runtime checks
  -> receive acceptance
  -> mock send acceptance
  -> optional real send acceptance
```

## Manual Acceptance

Run:

```bash
npm run build:web
.venv/bin/python -m compileall apps/api scripts/check_release_readiness.py
npm run db:migrate
npm run check:release
```

Then review:

1. `docs/FEISHU_SETUP.md` covers stream worker setup and Feishu acceptance.
2. `docs/WECOM_SETUP.md` covers public callback, trusted IP, and WeCom
   acceptance.
3. Both docs default to simulated external send before real send.
4. `npm run check:release` requires both docs.
