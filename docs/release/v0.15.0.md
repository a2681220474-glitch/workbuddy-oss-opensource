# WorkBuddy OSS v0.15.0 Feishu Production Loop

v0.15.0 moves the Feishu path from "manual real test" toward a formal local
production loop. The focus is not adding demo surface area; it is making the
real receive, approval, send, audit, and recovery path easier to operate.

## Changes

- Added a `feishu-worker` service to Docker Compose so API, Web, and the
  Feishu long-connection worker can be started as one local stack.
- Updated Feishu worker diagnostics with both local and Docker startup commands.
- Added Feishu production readiness checks for credentials, real adapter
  policy, worker heartbeat, callback encryption compatibility, recent message
  receive, and external send policy.
- Added receive requirements and production notes to the Feishu diagnostics API.
- Approval context now includes send preflight data and delivery history.
- Approval rows now expose delivery attempt count and latest delivery time.
- Failed approval deliveries are explicitly presented as retryable from the UI.
- Bumped local app/API/runtime version to `0.15.0`.

## Manual Acceptance

1. Restart API and Web after pulling this version.
2. Open `http://localhost:5173/#config` and confirm the Feishu runtime card
   shows both:
   - `npm run dev:feishu-stream`
   - `docker compose up feishu-worker`
3. Open `http://localhost:5173/#feishu` and confirm the `v0.15.0 正式运行检查`
   card appears.
4. Confirm the readiness checks clearly show:
   - Feishu credentials status,
   - real adapter status,
   - worker heartbeat status,
   - encryption warning if `FEISHU_ENCRYPT_KEY` is configured,
   - recent Feishu message status,
   - external send policy.
5. Run `npm run check:feishu-stream`.
6. Start the worker with either `npm run dev:feishu-stream` or
   `docker compose up feishu-worker`.
7. Send a message to the Feishu test bot and confirm the diagnostics page shows
   the recent message and business trace.
8. Open `http://localhost:5173/#approvals`, approve a Feishu draft, inspect the
   send preview, and send it.
9. If sending fails, keep the approval in approved/edited state and click
   `重试`; confirm the approval detail drawer shows multiple delivery attempts.

## Next Version

v0.15.1 should continue hardening the Feishu production path:

- Add encrypted Feishu callback support instead of only detecting the risk.
- Add structured worker error logs and a retry queue for failed receive events.
- Add interactive Feishu card approval for approve/edit/reject/send.
- Add scheduled SLA/task reminders that can notify Feishu after approval.
- Start introducing production auth/RBAC before widening team usage.
