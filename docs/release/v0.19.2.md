# WorkBuddy OSS v0.19.2 Knowledge Hit Tracking

`v0.19.2` continues the `v0.19.x` knowledge and RAG formalization line. This
version makes knowledge usage traceable by recording when a ticket matches a
published knowledge item.

## What Landed

- Added `knowledge_hits` table.
- Added Alembic migration `20260604_0003_knowledge_hits`.
- Ticket knowledge suggestions now record deduplicated knowledge hits.
- Knowledge item detail API now returns:
  - `hit_summary`
  - recent `hits`
  - hit events in the timeline
- Knowledge item detail drawer now shows:
  - total hits
  - average match score
  - latest hit time
  - recent source ticket/query and answer snapshot
- Version labels updated to `v0.19.2`.

## Manual Acceptance

1. Open `http://127.0.0.1:5173/#tickets`.
2. Open a ticket and use the knowledge suggestion workflow.
3. Open `http://127.0.0.1:5173/#knowledge`.
4. Open a KnowledgeItem detail drawer.
5. Confirm hit summary and recent hit records are visible.
6. Reopen the same ticket suggestion and confirm it updates the existing hit
   instead of creating duplicate hit rows.
