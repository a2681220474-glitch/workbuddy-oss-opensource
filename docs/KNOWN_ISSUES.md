# Known Issues

更新时间：2026-06-15
本地版本：`v1.1.16`
远程 ECS：`v1.1.14` 已完成真实人工验收

## Deployment

- 本机没有可用 Docker 环境，因此 Compose 全栈未在当前开发机实跑。
- ECS 已跑通过 Caddy、Web、API、runtime-jobs、PostgreSQL、Redis，并在用户授权后升级到 `v1.1.14`。
- 远程当前使用免费自动解析 HTTPS 域名 `https://workbuddy.example.com`；尚未绑定用户自有正式域名。
- 飞书卡片按钮回调地址已生成：`https://workbuddy.example.com/api/channels/feishu/webhook`。
- `v1.1.3` 已完成本地服务托管、自动重启、健康检查和日志轮转；远程 Compose 已有重启策略，但 systemd/主机级长期验收仍待 ECS 升级时执行。

## Security And Identity

- 本地登录、密码哈希、会话门禁和写操作 RBAC 已完成当前阶段。
- `v1.1.0` 已将运行密钥迁移到 Fernet 加密仓库；主密钥仍是本机文件，生产部署后续可接 KMS、Vault 或系统钥匙串。
- 当前 RBAC 以路由级角色边界为主，尚未细化到字段级或资源级授权。

- Docker/PostgreSQL/Redis 全栈和长期 worker 已在 ECS 运行；`v1.1.16` 已完成 PostgreSQL 备份恢复隔离演练。

## AI And Knowledge

- `v1.1.4` 已完成本地确定性向量、混合检索、引用片段、命中反馈和知识回滚闭环。
- 当前向量模型是适合私有化零成本运行的本地特征哈希，不等同于云端大模型 embedding；后续可按需接入可插拔模型。
- 真实模型调用仍依赖外部服务可用性、额度和本机网络证书环境。

## Connectors

- 飞书/企微已有历史真实接收和真实发送成功样本；`v1.1.5` 未自动触发新的真实外发。
- 真实测试发送现在必须同时开启全局外发开关、勾选确认并输入精确授权短语。
- 飞书长连接 worker 已支持本地显式托管；`v1.1.14` 新消息、北京时间、审批完成态和详情链接已完成人工验收。
- `v1.1.11` 已补齐飞书审批卡片按钮回调诊断；如果飞书提示“目标回调服务当前未在线”，需要配置公网 HTTPS 回调地址 `/api/channels/feishu/webhook`，长连接 worker 不能接收卡片按钮点击。
- `v1.1.11` 线上热修复已解决配置中心普通参数写入容器临时文件、部署默认值覆盖网页保存值的问题；API 与 Worker 现统一读取共享持久卷中的 `apps/api/data/runtime.env`。
- 当前 ECS 飞书 Worker 已连接并由 Compose `unless-stopped` 托管；健康接口显示 `running / ok`。
- 飞书 Worker 离线期间发送的历史消息不会自动补发；Worker 恢复在线后需要重新发送一条新消息进行接收验收。
- `v1.1.12` 已统一 API 北京时间输出，并修复飞书审批卡片完成态和详情链接。
- `v1.1.13` 已修复通过/拒绝成功后飞书仍提示 `200672` 的回调响应兼容问题。
- `v1.1.14` 已补齐飞书 SDK 要求的 `card.type=raw` 与 `card.data` 并受控部署；用户已于 2026-06-15 确认真实人工验收通过。
- `v1.1.15` 已将真实验收和远程 ECS 升级写入发布审计，连接器不再显示人工待验。
- `v1.1.16` 已完成 PostgreSQL 隔离恢复、临时库自动清理和正式库未变更复核；九项发布基线全部完成。
- 内容相同但飞书消息 ID 不同的多次真实发送会分别保留；只有同一消息 ID 的平台重试会被幂等忽略。
