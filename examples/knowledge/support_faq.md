# Demo Support FAQ

This lightweight knowledge file is for Phase 0 support-ticket demos only.

## Login Verification Code Error

Ask the customer to confirm phone/email, browser, approximate time and whether the code expired. Suggest retrying after clearing browser cache. If the issue blocks production usage, mark priority high.

## Refund Request

Do not promise a refund directly. Create a ticket, collect order ID and payment time, and route to human approval before replying externally.

## Invoice Correction

Collect company name, tax ID, invoice amount, old invoice number and corrected invoice title. Create a normal-priority ticket unless the customer says it blocks reimbursement or procurement.

## Import Failure

Ask for file type, row count, error screenshot and import time. Reassure that existing data is not modified unless the backend confirms partial writes. Escalate if the message mentions HTTP 500 or data loss.

## Missing Documentation

Create a knowledge gap note when a customer cannot find documentation for CSV/JSON import, approval queue, AgentRun audit or local deployment.
